## Headline Features

### App Bypass (split-exclude per app)

Apps can now be excluded from the VPN tunnel. Useful for banking,
WeChat, 12306, gaming, and other apps that flag VPN IPs.

- **Where:** Dashboard → Advanced Settings → "不走代理的应用"
- **Sources:** running processes (macOS / Windows / Linux desktop) or
  installed apps (Android, with regional auto-detection — China bundle
  preloads WeChat / Alipay / 12306 / banking apps)
- **Mechanism:**
  - macOS / Windows / Linux: rule-engine `process_name` / `package_name`
    match via new `ProcessSearcher` provider with shared cache + TTL
    eviction (k2 `511e8ec`, `9f1201c`, `649bb32`, `d64eb0d`)
  - Android: kernel-level `addDisallowedApplication` (Android sandbox
    blocks `/proc/net/tcp` so userspace matching can't work)
  - iOS: not supported (OS-level limitation)
- **Privacy:** app names never appear in debug logs (test-enforced).
- **UI guard:** Advanced Settings is locked while VPN is connected —
  previously settings could be saved but had no effect.

### Password Login (cookie + Bearer)

Full password authentication flow rolled out across all surfaces.

- **Backend:** new `POST /api/auth/web-login/password` (cookie-based, no
  device binding) alongside the existing `POST /api/auth/login/password`
  (device-bound). Both gate on `zxcvbn` strength ≥ 3 + 10-char minimum.
- **Anti-enumeration:** unknown user, wrong password, and "password
  not set" all collapse to `ErrorInvalidCredentials`. Only account
  lockout diverges (observable regardless).
- **Set / Change password:**
  - Email is fed to `zxcvbn` `userInputs` so passwords containing the
    user's email or local part are rejected.
  - Every successful password change sends a notification email
    (defense-in-depth against cookie/token theft — we trust the bearer).
- **Frontend (webapp + web):**
  - Password tab in EmailLogin (Purchase / gift / survey flows on web).
  - `PasswordStrengthMeter` with i18n score labels.
  - `/account/security` page (web) + Account screen entry (webapp) for
    set / change.
  - `hasPassword` flag on `DataWebLoginUser` keeps the CTA fresh after a
    set-password call.
- **Critical fix (today):** `webapp` `cloud-api.ts` allowlist now
  includes `/api/auth/login/password` — without it, login succeeded on
  the server, returned tokens, and the webapp silently threw them away.

### Router (k2r) Robustness — Bug 6-11 Bundle

Six gateway bugs caught during k2r field UAT:

| Bug | Fix |
|----:|-----|
| 6   | Preserve iptables on startup (`4aa829d`) |
| 7   | Bound infinite procd respawn retry (`5057f6d`) |
| 8   | Wait for `xtables.lock` with `-w 5` (`f36cf6e`) |
| 9   | Tear down iptables before `engine.Stop` (`825fd52`) |
| 10  | Plug dnsmasq DNS leak via OUTPUT REDIRECT (`0b7bf03`) |
| 11  | `install-k2r.sh` preflights kernel TPROXY support (`2640b1c`) |

Plus: idempotent `iptables` Install/Remove robust to stale rules, retry
auto-reconnect with bounded backoff, state.json preserved across
daemon-internal shutdown, gateway iptables errors reclassified as 412
client (not 570 server).

### k2 Wire Protocol Hotfix

Two production-impact fixes already deployed on v0.4.5 hotfix server
side, now bundled into the client release:

- **Server-side:** drop UDP frames when server session `readCh` is full
  to prevent client stall (`ab435b1`).
- **Client-side:** bound UDP mux `writeFrame` with deadline + dead flag
  (`907f724`). Eliminates the long-session UDP cap stall pattern.

## Bug Fixes

### k2 Engine / Provider

- **Win UDP wildcard attribution** (`7d7f785`) — Windows kernel
  records `net.Dial("udp")` / QUIC / game sockets as `0.0.0.0:port`;
  exact `LocalAddr` matching silently dropped them all.
- **Linux fresh-cache miss** (`587fc34`) — `ProcessSearcher` rebuilds
  inode index on cache miss; stale-while-revalidate previously dropped
  newly-launched processes.
- **macOS lsofLookup self-PID skip** (`82025a9`) + **wire
  `DarwinProcessSearcher` into engine.Config** (`3c56a62`) — two
  silent bugs that disabled macOS `process_name` attribution.
- **rule:** meta-only routes take priority over DNS tmp cache
  (`83c4510`).
- **core:** preserve `process_name` match through tunnel routing
  (`7649164`).
- **engine perf:** skip `ProcessSearcher` when no rule references it
  (`3257a21`).
- **provider:** explicit `Unlock` in `processCache.set` (`9c1ebbd`).
- **daemon:** suppress `closeTunnel` teardown noise during `doUp` to
  eliminate webapp UI flicker (`785315c`).

### Auth

- **webapp:** `/api/auth/login/password` now in `AUTH_TOKEN_PATHS`
  allowlist — fixes silent password-login token discard (`5a985a48`).
- **api:** anti-enumeration consistency for `api_password_login`.
- **api:** account-locked + no-password-set cases covered by tests.

### Other

## Upgrade / Migration Notes

- macOS NE-mode (system extension) is gone. Users upgrading from any
  prior beta with NE installed will fall through to daemon mode — the
  daemon installer pkg flow stays the same.
- `addDisallowedApplication` requires `BIND_VPN_SERVICE` (already
  granted). No new permission prompt.
- `installed apps` query on Android 11+ requires `<queries>` element —
  added. Without it, the manifest would silently return an empty list.
- `install-k2r.sh` now preflights TPROXY support — kernels without
  `xt_TPROXY` will fail fast with a clear message instead of failing
  later at iptables `INSTALL`.
