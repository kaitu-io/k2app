## 验证步骤

### 1. 部署

```bash
git push origin main
# 等待测试环境部署完成
# 或者本地：
cd api/cmd && go build -o center . && ./center start -f -c ../config.yml
cd web && yarn dev
```

### 2. 执行顺序

1. **先跑 A01–A07**（RBAC 超管流程） — 如果这些不通，B/C 模块都无法正常访问
2. **再跑 B01–B07**（License Key 管理端） — 需要超管权限
3. **然后 C01–C08**（分销商等级） — 需要超管权限
4. **再跑 B08–B16**（License Key 兑换端） — 需要非管理员测试账号
5. **最后 A08–A12**（RBAC ops 用户） — 需要用 A04 设置好的 ops 账号登录

### 3. 测试账号准备

| 角色 | 准备方式 |
|------|---------|
| 超级管理员 | 已有（isAdmin=true） |
| Ops 只读员工 | A04 步骤中设置 ops_viewer 角色 |
| 免费用户（从未付费） | 注册新邮箱，不购买 |
| 付费用户 | 已有付费账号 |
| 分销商 | 已有或通过用户详情页开启 |

### 4. 判定标准

- **P0 全通过** → 可发布
- **P1 有失败** → 记录 issue，评估是否 block 发布
- **P0 有失败** → 必须修复后重测
