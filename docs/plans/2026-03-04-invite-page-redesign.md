# 邀请页面交互重构设计

## 背景

当前邀请页面存在以下问题：
1. "生成完整内容"、"生成链接"、"生成新验证码"三个按钮让用户困惑
2. 生成链接需要先选有效期（弹出 popover），操作太深
3. 复制后没有清晰提示有效期和分享建议
4. 分享文案包含"开途"、"VPN"等敏感词，在微信中有法律风险

## 设计原则

- 按**使用场景**分区，不按操作分按钮
- 每个场景一个卡片，一个按钮，零决策
- 桌面/手机差异化：桌面用 QR 码辅助手机扫码分享，手机用系统分享
- 分享文案模糊化，不提产品名和 VPN

## 页面结构

### 顶部：邀请码信息

保留现有的邀请码展示区，简化：
- 邀请码（点击复制）
- 已注册 N 人 / 已付费 N 人
- 备注（可编辑）

### 区域一：分享给好友（7天短链）

**桌面端：**
```
💬 分享给好友

┌──────────────────────────────────────────┐
│                                          │
│         ┌──────────┐                     │
│         │ QR Code  │  用手机扫码          │
│         │ (7天短链) │  直接转发给好友       │
│         └──────────┘                     │
│                                          │
│  ⏰ 链接7天内有效，适合在聊天中分享         │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │ 📋 复制分享内容                    │    │
│  └──────────────────────────────────┘    │
│                                          │
└──────────────────────────────────────────┘
```

**手机端：**
```
💬 分享给好友

┌──────────────────────────────────────────┐
│                                          │
│  ⏰ 链接7天内有效，适合在聊天中分享         │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │ 📤 分享给好友                      │    │
│  └──────────────────────────────────┘    │
│                                          │
└──────────────────────────────────────────┘
```

**交互细节：**
- 页面加载时自动调 API 获取 7 天短链（`/api/invite/my-codes/{code}/share-link?expiresInDays=7`）
- 桌面：QR 码自动生成，展示 7 天短链；"复制分享内容"复制模糊文案
- 手机：点击"分享给好友"→ `navigator.share()` 系统分享面板
- `navigator.share` 不可用 → 降级为"📋 复制分享内容"
- 复制后 toast：「已复制，可粘贴到微信发送给好友」

**分享文案（模糊化，不提产品名/VPN）：**
```
🎁 送你一份好礼，打开链接领取
https://s.kaitu.io/xxxxx
```

### 区域二：推广链接（永久有效）

桌面端和手机端完全一致：

```
🔗 推广链接（永久有效）

┌──────────────────────────────────────────┐
│                                          │
│  https://www.kaitu.io/s/ABCD1234             │
│                                          │
│  适合社交媒体简介、博客、视频描述           │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │ 📋 复制链接                        │    │
│  └──────────────────────────────────┘    │
│                                          │
└──────────────────────────────────────────┘
```

**交互细节：**
- 永久链接格式：`https://www.kaitu.io/s/{CODE}`，前端直接拼接，不调 API
- 不过期，无 QR 码
- 复制后 toast：「推广链接已复制」

### 区域三：奖励规则

保留现有 `InviteRule` 组件，无变化。

### 区域四：分销管理（仅经销商可见）

保留现有 `RetailerStatsOverview` 组件。

**"生成新邀请码"按钮**从主页面移除，仅在"查看全部"列表页（`/invite-codes`）中对经销商显示。

## 删除/简化的内容

| 当前 | 改为 |
|------|------|
| "生成完整内容"按钮 + 有效期 popover | "复制分享内容"一键操作（固定 7 天） |
| "生成链接"按钮 + 有效期 popover | 删除（合并到上面） |
| "生成新验证码"按钮 | 从主页面移除，仅经销商列表页保留 |
| ExpirationSelectorPopover 组件 | 主页面不再使用（列表页可保留） |
| 4 种有效期选项 (1/7/30/365) | 主页面固定 7 天 + 永久链接分开 |
| 含产品名的分享文案 | 模糊化："送你一份好礼" |

## 需要的代码变更

### 修改的文件
- `webapp/src/pages/InviteHub.tsx` — 主页面重构
- `webapp/src/hooks/useInviteCodeActions.ts` — 简化分享方法，更新文案
- `webapp/src/i18n/locales/*/invite.json` — 新增/修改 i18n 键（7 个 locale）

### 可能删除/不再使用的
- `webapp/src/components/ExpirationSelectorPopover.tsx` — 主页面不再需要（列表页如果仍用可保留）

### 已确认
- 永久链接格式：`https://www.kaitu.io/s/{CODE}` — 直达落地页，已实现在 `web/src/app/[locale]/s/[code]/page.tsx`
- 7天短链格式：以 API 返回的 `shareLink` 字段为准（走 Unred 防红服务），前端不硬编码短链域名
- 永久链接不走 Unred，前端直接拼接，无需 API 调用
