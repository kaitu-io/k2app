## Dimension 1 — Title & Subtitle (5/10)

### Current state

| Field | zh-Hans | en-US |
|---|---|---|
| Name | 开途 - 越拥堵，越从容 (18 chars) | Kaitu - Network Accelerator (27 chars) |
| Subtitle | 安全快速的网络加速器 (10 chars) | Fast, Secure & Private Proxy (28 chars) |

### Issues

**zh-Hans:**
- 🟢 "越拥堵，越从容" 是**世界级 tagline** — 直接来自 context 的定位。保留。
- 🟡 Name 吞掉了 tagline → 最强定位句卡在不可点的 name 字段，不是搜索可见的 subtitle。
- 🔴 Subtitle "安全快速的网络加速器" 泛化；每家竞品都这么写。零差异化。
- 🔴 Subtitle 完全浪费 k2cc / 抗 GFW 定位。kaitu.io 首页 hero 的 "别人断线，你满速" 是被证明过的句子，没进 listing。

**en-US:**
- 🟡 "Network Accelerator" 是中国区合规绕词 — 全球店没必要。美区等 VPN 合法区直用 "VPN" 搜索量更高。
- 🔴 "Fast, Secure & Private Proxy" 和每家竞品一模一样。零差异化。

### Fixes（已按 brand-naming 2026-04-21 决策对齐：中国 = 开途 / 海外 = Overleap）

**zh-Hans（30 字符限）：**
```
Name:     开途VPN (6 chars)
Subtitle: 别人断线，你满速 · 抗拥堵加速 (15 chars)
```
Rationale: tagline 从 name 挪进 subtitle（搜索可见），name 留给 brand + category（"开途VPN" —— 注意 brand-naming 也指这是传播层首选词）。subtitle 兼顾定位 + 关键词源（"抗拥堵" "加速" "VPN"）。

**Alt zh-Hans（若合规要求继续回避 "VPN"）：**
```
Name:     开途 网络加速
Subtitle: 越拥堵越从容 · 全球节点加速
```

**en-US（全球店 — Overleap 品牌）：**
```
Name:     Overleap VPN (12 chars)
Subtitle: Line-rate under packet loss (27 chars)
```
Rationale: "Overleap VPN" 是品牌 + 品类干净组合（无商标冲突、无 kaitai 纠错问题）。Subtitle 用 context 原句 "line-rate under packet loss" 作核心卖点，指向 k2cc 基准数据。

**备选 en-US subtitle**（若想强化反审查定位）：
```

```
适合海外隐私向受众（Mullvad / IVPN 赛道），低竞争关键词 "stealth VPN" 精准匹配。

## Dimension 4 — Ratings & Reviews (2/10)

### Current state

- **中国大陆 App Store：5.0 星，4 个评价**
- Japan / Taiwan / US: "insufficient ratings to display"
- 其他 locales: 无评价
- 无开发者回应（没东西可回）

### 为什么关键

~100 以下评价，App Store 搜索排名被重度压制（无论 metadata 质量）。评价量在低位是比关键词更强的排名信号。**从 4 → 100 评价比其他任何单项 ASO 动作价值都高**。

### Rating-startup plan（Kaitu 专属）

**原则：只提示已成功使用过的用户。** 太早提示 = 1 星评价。对 VPN 而言 "成功" 指 "连接成功 AND 真实用了这个连接工作"。

**触发条件**（in-app via `SKStoreReviewController`，全部满足才弹）：

| 条件 | 原因 |
|---|---|
| 用户已登录（有账号） | 白嫖用户不是评价来源 |
| 完成过 ≥3 次独立连接 session | 过滤误试即弃 |
| 最近 14d 累计连接时长 ≥ 60 min | 真实使用，不是装完不管 |
| 当前 session 干净断开（非错误断开） | 不在糟糕体验后弹 |
| 最近 90 天没弹过 | Apple 限制 3 / 365d |
| 最近 3 天无该用户支持工单 | 不爽用户在反馈流，不在评价流 |

**弹出位置**：干净断开长连接 session（>10 min）后。**不**在应用启动时弹，**不**在支付后弹。

**文案**（zh-Hans，iOS 原生 prompt 无法定制，但 pre-prompt 可定制）：

Pre-prompt banner（可忽略），干净断开后 2 秒显示：
```
标题: 觉得开途用着顺手吗？
正文: 一个五星评价对我们意义很大 —— 尤其在新版本推出时。
按钮: [评价一下] [以后再说]
```

用户点 [评价一下] → 触发 `SKStoreReviewController.requestReview()`
用户点 [以后再说] → 60 天内不再弹

**负面路径 — 反馈捕获：**
用户在评价 prompt 附近看到 "哪里用得不顺？" 链接（触发负面操作），路由到支持工单系统而非 App Store。把 would-be 1 星评价转成你能修的工单。

**预期轨迹：**
- Week 1–2: 埋点触发 + 加 pre-prompt UI
- Week 3–8: 100% 放量。按 10-15% DAU/周 弹（尊重 90d 冷却）。目标 2-3% prompt→rating 转化。若 DAU 1,000，~20-30 评价/周。
- Month 3: 主 locale ≥100 评价。排名提升可测量到。

**P1：立即上 2 个 in-app events**

- Event 2: "GFW 升级期间稳定连接"（reactive，网络断裂事件期间发） — 对接 `kaitu-growth` 的 GFW Event Response playbook

**P2：90 天目标建 3 个 Custom Product Pages**
1. `/cpp/family` — family JTBD 优先，家庭买家 persona，5 设备截图置前
2. `/cpp/developer` — k2s / 协议文档优先，技术派 persona，协议对比截图置前
