v0.4.3 重点完善了 iOS Always On VPN、四场景路由模型、会员等级体系、Linux 单二进制安装与连接质量反馈，并修复了大量稳定性与 UI 问题。同时为下一阶段的智能选服、节点评分、点对点直连与 Kaitu Router 完成了底层基础建设。

## New Features
- **iOS Always On VPN**: iOS 设置中可启用 Always On 模式，开机即自动连接，断开后系统自动重连
- **四场景路由模型 v3**: 翻墙 / 全局 / 自定义 / 直连四种场景，配合 MaxMind GeoIP 自动判断当前所在国家
- **ModeChip + TravelBanner**: 状态栏直观显示当前路由模式，出国旅行时自动提示切换合适的模式
- **会员等级 + 配额**: 不同会员等级对应不同设备数 / 路由 / 旅行配额，购买页与账户页完整展示
- **连接质量反馈**: 断线时弹出反馈对话框，自动收集日志；用户可对节点连接质量打分
- **Linux 单二进制安装**: Linux 桌面端不再使用 Tauri，改用嵌入了 webapp 的单 Go 二进制，`curl -fsSL https://kaitu.io/i/k2 | sudo bash` 一键安装
- **k2 CLI**: `k2 up <k2subs://...>` 直接连接订阅链接；多出站路由配置（`routes: [{via, match}]`）支持混合路由
- **英文品牌 Overleap**: 英文站正式启用 Overleap 品牌（中文继续使用「开途」）

## Bug Fixes
- **修复 iOS 断开后无法手动连接**: 断开时未清除系统的 onDemandRules，导致 Always On 状态下用户无法主动断开（ANC-13）
- **修复 Android 登录对话框意外关闭**: 输入邮箱 / 验证码时键盘弹出导致登录窗口关闭、键盘回收
- **修复 Android gomobile 兼容性**: 适配 anet v0.0.5，避免编译失败
- **修复订阅请求 goroutine 泄漏**: Subscription HTTP 客户端在 F2 场景下泄漏 goroutine，长期运行后耗尽资源
- **修复 wire UDP 僵尸连接**: 空闲会话回收时未通知 UDP 连接，导致僵尸连接
- **修复 IPv4-mapped IPv6 写入失败**: handleUDPDirect 与代理 UDP ReadFrom 改用 udp4
- **修复连接互斥锁饥饿**: TCP 与 QUIC 拨号添加内核级硬超时，避免 connectMu 饥饿
- **修复 wire 重置时 stream slot 泄漏**: 区分 EOF 与异常退出，避免槽位耗尽
- **修复 errgroup 清理只关闭部分 TM**: 引擎启动失败时正确关闭所有传输管理器
- **修复无 k2v5 出站时 TUN 模式空指针**: 拒绝无效配置而非崩溃
- **修复 webapp 抗封锁日志噪音**: antiblock CDN 探测日志降级
- **修复未登录用户访问反馈页时弹登录框**: 反馈页不再在未登录时尝试拉取工单
- **修复授权码全局唯一冲突**: 改为按批次唯一，不同批次可使用相同码值
- **修复邀请奖励唯一索引阻塞多受邀者**: 同一邀请人对多个受邀者的奖励正确入账
- **修复地理检测失败时无默认国家**: 检测失败回退到 CN
- **修复路由模式中残留的 home / home_proxy 预设**: 已下线的预设不再出现在选项中
- **修复"敬请期待"标签残留**: 已发布的路由模式选项移除 coming soon 标签

## Improvements
- **零停机出站替换**: SwappableDialer 支持运行中替换出站节点，无需断开重连
- **DNS Happy Eyeballs**: DNS 传输层并行竞速 QUIC stream + TCP-WS，提升弱网解析速度
- **桌面端死锁自愈**: 60 秒死锁检测，触发后清理 DNS 状态并退出，由系统服务管理器自动重启
- **订阅故障切换**: engine.Start 失败时自动尝试订阅返回的下一个服务器
- **Linux systemd 安全加固**: 服务单元启用 NoNewPrivileges、ambient CAP_NET_ADMIN+CAP_NET_RAW、失败自动重启；服务名统一为 kaitu.service，避免新旧安装路径冲突
- **k2b 规则库 v2**: 新增 exclude 域名支持与合并集合 schema；k2rule 升级到 v0.6.5（默认 cn_whitelist）
- **反馈工单含 git commit**: 客户端反馈自动附带版本与 commit，便于精准定位
- **webapp 7 语种 i18n 完善**: 错误处理与文案补齐
- **认证 / 安全加固**: 修复 auth TOCTOU、WS 写入互斥、UDID 校验、auth 超时、API 重定向校验等多项问题

## Infrastructure

以下功能已完成代码与基础设施建设，但暂未对终端用户开放，将在后续版本逐步上线。

- **智能选服后端**: k2subs:// 订阅协议、按权重选服、故障切换、SmartServerSelector UI（智能选择 / 指定服务器 / 自部署 三 Tab）、按国家筛选与冷启动恢复
- **节点推荐评分**: probe.Registry 内存缓存、background probe.Service、recommendScore 统一评分、RecommendBar 节点分数条、ProbeChip 实时探测状态、Dashboard 周期性重测（5 分钟）
- **k2p 点对点直连**: 全新 P2P 架构 — STUN 探测、NAT 类型识别、双向打洞、对称 NAT 生日攻击、ICE 候选交换、TURN relay 兜底、peer auth (bcrypt)、QUIC over ICE、signal 服务器
- **Kaitu Router (k2r)**: OpenWRT 软路由产品线 — gateway daemon、HTTP API、CLI 入口、TPROXY 拦截、nftables/iptables 三层降级、LAN 设备发现 (ubus/dnsmasq/ip-neigh)、MAC 白名单、按 Tier 配额、OTA 升级、加密 KV 存储、嵌入式 webapp、一键安装脚本
- **会员四档模型 (tier rename)**: tier 从 `pro` 重命名为 `basic` 并引入四档骨架 `lite` / `basic` / `family` / `business`；配额（设备数 / 路由器 / LAN 接入）由 `TierQuotas` 常量派生，`User` / `Plan` 的 `max_device` / `max_router_device` / `max_lan_client` 列已删除（JSON 输出通过 `MarshalJSON` 注入，老客户端零破坏）；代付功能（forUsers / forMyself）彻底下线；购买校验：首次任意档位放行、续费必须同档；新增公开 `GET /api/tiers`、admin `GET /app/tiers` / `PUT /app/users/:uuid/tier`（含审计日志）；前端购买页按 tier 过滤 plans，manager 新增档位 dropdown + 配额预览卡 + 用户改档 Dialog。当前生产仅开放 basic 档，其余三档的商业化/营销作为后续版本逐步上线。

## Breaking Changes
- **k2 CLI 移除命令**: `k2 connect` → 改用 `k2 up <k2subs://...>`；`k2 android-install` → 改用守护进程 `/api/helper` API 或 MCP 工具；`demo-config` 别名 → 改用 `k2 config show`
- **路由配置格式**: 客户端配置从 Server+Rule 重构为 `routes: [{via, match}]` 多出站数组（webapp / 守护进程已自动迁移，外部脚本调用方需更新）
