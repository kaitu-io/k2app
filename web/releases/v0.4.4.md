**v0.4.4 是 Kaitu Router (k2r) 专属版本** —— 桌面 / 移动端继续使用 v0.4.3，下个 desktop / mobile 发布请关注 v0.4.5。本版完成 Kaitu Router 正式发布、把 v0.4.3 完成的「智能选服后端」上线为 Auto 自动选服功能，并带来一批安卓 / webapp 稳定性修复、`-tags release` 生产构建加固，以及 k2 引擎层多个并发与资源泄漏修复。

路由器安装：`wget -qO- https://kaitu.io/i/k2r | sudo sh`（脚本自动检测 ARM64 / x86_64 / ARMv7 三种架构）。

## New Features
- **Auto 自动选服**: Dashboard 顶部新增 Auto 卡片，连接时按 recommendScore 在 Top-5 节点中随机挑选；v0.4.3 的「智能选服 Infrastructure」此版正式对终端用户开放
- **K2subs 国家选择 UI**: SmartServerSelector 的「智能选择」Tab 增加 Auto 与按国家筛选；用户可固定走某国家的订阅出口
- **Kaitu Router (k2r) 正式发布**: 网关守护进程支持解析 k2subs:// 订阅、k2 出站、TPROXY 拦截、按 Tier 配额、OTA 升级；OpenWrt / 软路由 / Linux SBC 一键安装 `wget -qO- https://kaitu.io/i/k2r | sudo sh`
- **安装页路由器选项**: `/install` 增加「路由器」面板，引导 OpenWrt / 软路由 / Linux SBC 用户安装 k2r
- **Admin 设备统计按 Router/App 拆分**: 管理后台「使用统计」分别展示路由器接入与 App 接入的设备数

## Bug Fixes
- **修复 Android VPN 自回环**: K2Plugin 通过 `addDisallowedApplication` 排除自身 UID，避免应用自身流量绕回 TUN
- **修复 Android 断开 UI 卡顿 (#254)**: 解耦断开 UI 反馈与引擎拆除时序，UI 立即响应、底层异步收尾
- **修复旧 WebView 缺少 crypto API**: webapp 增加 `crypto.randomUUID` 与 `subtle.digest` polyfill，老安卓 / 老 iOS WebKit 不再报错
- **修复 Auto 选服后状态不刷新**: 连接成功后轮询 VPN 状态、刷新 tunnel 缓存；analytics 改上报 Auto 解析后的 ipv4
- **修复 enrichFromTunnelList 覆盖用户选择**: 后台拉取 tunnel 列表时不再误改 `selectedCloudTunnel`
- **修复 mcp machineid 跨平台兼容**: 替换 `denisbrodbeck/machineid` 为按 OS 实现的 hwid 模块，避免 macOS / Windows 取值差异
- **修复 wire UDP 会话泄漏**: 引擎层修复空闲会话回收时的 UDP 连接残留
- **修复规则引擎与 DNS 路径 panic**: rule / DNS 模块多个边界条件 panic 修复
- **修复 Windows pipe-test 偶发卡死**: CI 上 Windows pipe 测试 flake 修复
- **修复 engine.fail() 孤儿 healthMonitor**: 引擎进入 fail 态时正确停掉 healthMonitor，避免孤儿 goroutine
- **修复 gateway URL 解析与 engine 对齐**: 网关侧改用 `wire.ParseURL`，与引擎统一解析行为

## Improvements
- **生产构建统一加 `-tags release`**: 所有发布产物剥离 E2E helpers / 测试桩，减少二进制体积、收紧攻击面；CI 与 Makefile 全线统一
- **抽出共享 subscription 包**: 桌面 daemon 与 OpenWrt gateway 共用 `subscription.Manager` / `Resolve` / `ApplyResolved`，避免重复实现
- **k2subs E2E 测试套**: 新增 A-E 共 5 组端到端测试，覆盖 pick 层、probe 联动、refresh 联动、生命周期/并发、真实用户场景
- **`make dev-openwrt`**: 新增跨平台 OpenWrt 网关本地开发命令（含 cross-platform shasum），一键拉起 dev 网关
- **wire smux 锁安全**: smux sessionsMu 改用 `deadlock.Mutex` 便于死锁审计

## Infrastructure
- **会员四档商业化**: tier 已切换到 `lite` / `basic` / `family` / `business` 四档骨架（v0.4.3 完成），生产仍仅开放 basic；其余三档的购买页与 EDM 推广将在后续版本逐步上线
- **k2p 点对点直连**: STUN/ICE/TURN 已就绪（v0.4.3 完成），生产暂未启用，等待 signal 服务器灰度
