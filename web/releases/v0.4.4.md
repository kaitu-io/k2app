**v0.4.4 是 v0.4.3 之后桌面 / 移动 / 路由器三端的统一发布**，重点包括：把 v0.4.3 完成的「智能选服后端」上线为 Auto 自动选服功能、Kaitu Router (k2r) 产品线正式发布、登录与认证体系全面加固（Cookie/Bearer 双轨 + 邮箱验证码生命周期 + Sentry 监控）、官网 /releases 多平台下载与浏览器兼容提示、以及 k2 引擎层多个并发与资源泄漏修复。

路由器安装：`wget -qO- https://kaitu.io/i/k2r | sudo sh`（脚本自动检测 ARM64 / x86_64 / ARMv7 三种架构）。

## New Features
- **Auto 自动选服**: Dashboard 顶部新增 Auto 卡片，连接时按 recommendScore 在 Top-5 节点中随机挑选；v0.4.3 的「智能选服 Infrastructure」此版正式对终端用户开放
- **K2subs 国家选择 UI**: SmartServerSelector 的「智能选择」Tab 增加 Auto 与按国家筛选；用户可固定走某国家的订阅出口
- **Kaitu Router (k2r) 正式发布**: 网关守护进程支持解析 k2subs:// 订阅、k2 出站、TPROXY 拦截、按 Tier 配额、OTA 升级；OpenWrt / 软路由 / Linux SBC 一键安装 `wget -qO- https://kaitu.io/i/k2r | sudo sh`
- **安装页路由器选项**: `/install` 增加「路由器」面板，引导 OpenWrt / 软路由 / Linux SBC 用户安装 k2r
- **`/routers` 4 步 DIY 安装向导**: 路由器产品独立落地页，按步骤引导从下载固件到完成连接
- **`/releases` 多平台下载**: 版本历史页直接列出 macOS / Windows / Linux / Android / iOS 全平台安装入口
- **Linux arm64 桌面端发布**: Linux 桌面单二进制开始同时发布 arm64 与 amd64 架构（树莓派 / ARM 笔电可用）
- **Admin 设备统计按 Router/App 拆分**: 管理后台「使用统计」分别展示路由器接入与 App 接入的设备数
- **断开评分 ambient 底部条带**: 断开后评分入口改为页面底部的轻量条带，不再用打断式 Dialog
- **BrowserWarningBar**: 在主流浏览器之外（旧版 / WebView / 微信内置浏览器）打开 kaitu.io 时显示友好提示，并把 WeChat-Android 购买兜底统一收敛到这个组件
- **旧版 iOS 设备警告**: iOS 系统版本低于 JS 基线时显示警告并屏蔽 Sentry 噪音
- **礼品码兑换内嵌登录**: 未登录访问礼品码兑换页时直接内嵌登录表单，无需跳转再回跳
- **Sentry 错误监控 + Session Replay**: 官网接入 Sentry，重要错误自动上报并可回放用户会话辅助定位
- **反馈页 FAQ 优先布局**: `/feedback` 头部直接列出 9 条常见问题（含 Adobe CC 等应用兼容、节点选择、AllNationConnect 老用户引导），未解决再开工单；登录用户可一键跳转「我的工单」
- **代付人设置页**: 旧「代付成员管理」服务端已下线（PROXY_MEMBERS_DEPRECATED），替换为简洁的「代付人设置」流程（设置 / 查看 / 移除代付人）

## Bug Fixes
- **修复 Android VPN 自回环**: K2Plugin 通过 `addDisallowedApplication` 排除自身 UID，避免应用自身流量绕回 TUN
- **修复 Android 断开 UI 卡顿 (#254)**: 解耦断开 UI 反馈与引擎拆除时序，UI 立即响应、底层异步收尾
- **修复旧 WebView 缺少 crypto API**: webapp 增加 `crypto.randomUUID` 与 `subtle.digest` polyfill，老安卓 / 老 iOS WebKit 不再报错
- **修复 Auto 选服后状态不刷新**: 连接成功后轮询 VPN 状态、刷新 tunnel 缓存；analytics 改上报 Auto 解析后的 ipv4
- **修复 Auto 选服误用黑名单节点**: pickAutoTunnel 现在严格跳过 score=0 的硬黑名单节点
- **修复 standalone Linux 状态映射**: standalone-k2 模式下 status 响应正确归一化 disconnected / error
- **修复 enrichFromTunnelList 覆盖用户选择**: 后台拉取 tunnel 列表时不再误改 `selectedCloudTunnel`
- **修复 tunnel 早期生命周期评分偏低**: 修复 tunnel 上线前几个周期 recommendScore 被低估的偏差
- **修复 Center 邮箱验证码生命周期**: 重构发送 / 校验 / 过期逻辑，杜绝穷举与回收异常
- **修复 Center auth 安全问题**: 修复 log 泄漏、panic 路径与 OTP 随机性，整体收紧 Center 鉴权
- **修复 Safari / iOS Cookie 被阻断时登录失败**: 自动回退到 localStorage Bearer Token；并吞掉 `localStorage.SecurityError` 不再黑屏
- **修复 admin 设备列表显示 email hash**: `/manager` 活跃设备页现在显示 UDID（哈希仅用于内部索引）
- **修复 admin 设备统计错误吞掉**: 后台 GORM 错误未检查导致统计静默归零
- **修复 EDM 批量发送被慢翻译阻塞**: AI 翻译加 30s 硬超时，避免单条卡死整批
- **修复 kaitu.io 根路径 CloudFront 缓存污染**: 不再误把 zh-CN 用户重定向到 en-GB
- **修复 iOS 上 Chrome / Firefox / Edge / Opera 被误判为 WebView**: 浏览器家族识别允许列表修正
- **修复 mcp machineid 跨平台兼容**: 替换 `denisbrodbeck/machineid` 为按 OS 实现的 hwid 模块，避免 macOS / Windows 取值差异
- **修复 wire UDP 会话泄漏**: 引擎层修复空闲会话回收时的 UDP 连接残留
- **修复规则引擎与 DNS 路径 panic**: rule / DNS 模块多个边界条件 panic 修复
- **修复 Windows pipe-test 偶发卡死**: CI 上 Windows pipe 测试 flake 修复
- **修复 engine.fail() 孤儿 healthMonitor**: 引擎进入 fail 态时正确停掉 healthMonitor，避免孤儿 goroutine
- **修复 gateway URL 解析与 engine 对齐**: 网关侧改用 `wire.ParseURL`，与引擎统一解析行为
- **修复网络抖动期间频繁重连**: k2 引擎对快速抖动 (flapping) 的网络事件加 debounce
- **修复规则库下载失败时硬 fail**: 下载失败时回退到磁盘已有的规则缓存，而不是阻塞启动
- **修复老 Android WebView 白屏**: Vite 6 默认 target 输出的 `??` / `?.` 语法在无 GMS 的国产安卓 (Huawei/Xiaomi/OPPO/Vivo) 自带 WebView (Chrome 51–75) 上抛 SyntaxError，导致 webapp 启动失败；编译目标降到 chrome70/safari14/firefox78/edge88 自动转译
- **修复代付页删除确认在 App 内无响应**: Capacitor 默认 WebView 在 iOS/Android 静默吞掉 `window.confirm`，点删除按钮无反应；统一改用 MUI Dialog 与项目其他确认弹窗保持一致

## Improvements
- **生产构建统一加 `-tags release`**: 所有发布产物剥离 E2E helpers / 测试桩，减少二进制体积、收紧攻击面；CI 与 Makefile 全线统一
- **生产构建默认 `K2_BUILD_LOG_LEVEL=info`**: 发布版默认走 info 级日志，降低日志噪音与磁盘占用（runtime SetLogLevel 仍可覆盖）
- **登录 Cookie 失败兜底**: Web 登录在 Cookie 写入失败时把 `accessToken` 写入 localStorage 作为 Bearer 兜底，保障 Safari 隐私模式 / 第三方 Cookie 屏蔽场景可登录
- **Web 登录回路重构**: Center auth 链路重构（移除 waymaker 中转 + tunnel score 统一 + 新增端到端登录测试），登录与选服路径更稳更可观测
- **OpenConnect (k2-oc) 协议下线**: webapp / api / 桌面 / 网关全链路移除 k2-oc，统一走 k2v5 / k2p
- **OSV-Scanner CI 工作流**: 新增依赖 CVE 持续扫描；同时一次性 patch 了 web / kaitu-center / 根工作区 67 个已知 CVE
- **抽出共享 subscription 包**: 桌面 daemon 与 OpenWrt gateway 共用 `subscription.Manager` / `Resolve` / `ApplyResolved`，避免重复实现
- **k2subs E2E 测试套**: 新增 A-E 共 5 组端到端测试，覆盖 pick 层、probe 联动、refresh 联动、生命周期/并发、真实用户场景
- **`make dev-openwrt`**: 新增跨平台 OpenWrt 网关本地开发命令（含 cross-platform shasum），一键拉起 dev 网关
- **wire smux 锁安全**: smux sessionsMu 改用 `deadlock.Mutex` 便于死锁审计
- **工单噪音收敛**: 自动生成的 `[Auto]` 工单默认静默，admin / MCP 工具新增过滤开关
- **邀请码列表分享按钮区分私享与公开链接**: 列表页的「分享」按钮加锁形图标 + 「Private Share」文案 + 悬浮说明 7 天有效；旁边的「复制链接」按钮明确仅复制公开推广链接 `/s/{code}`。两个按钮去掉过期时间弹窗，与 InviteHub 行为对齐
- **k2s DNS UDP 走 racingResolver**: 服务端 DNS UDP 处理统一走 racingResolver + `K2_DNS_SERVERS` 环境变量配置，提升解析稳定性
- **iOS App Store 版本号统一为三段**: 由历史「去掉前导 0」拼装规则改为「把 0 替换为 4」，0.4.4 → 4.4.4，与桌面 / Android 三段式对齐
